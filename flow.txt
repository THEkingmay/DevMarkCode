[1. Authentication Process (NextAuth + GitHub + Supabase)]
---------------------------------------------------
1. User clicks "Login with GitHub" button.
2. NextAuth redirects User to GitHub Authorization page.
3. User authorizes the app.
4. GitHub redirects back to callback URL (/api/auth/callback/github).
5. NextAuth (SignIn Callback):
   |
   +--> Check if User exists in Supabase (check by email or provider_account_id).
   |
   +-- [CASE: New User] (First time login)
   |    1. Retrieve profile data from GitHub (Name, Email, Avatar, GitHub ID).
   |    2. INSERT new row into 'users' table in Supabase.
   |    3. Create session.
   |
   +-- [CASE: Existing User]
        1. UPDATE user information (Avatar/Name) if changed (Optional).
        2. Create session.

[2. Create Post (Adding Knowledge/Snippets)]
---------------------------------------------------
1. User clicks "Create New Post".
2. User inputs:
   - Title
   - Content / Code Snippet
   - Link (Optional)
   - Tags (e.g., "React", "Bug", "Tutorial")
3. System validates input.
4. Database Transaction:
   |
   +--> 1. INSERT post data into 'posts' table (linked to current user_id).
   |
   +--> 2. Process Tags:
           For each tag in input:
           - Check if tag exists in 'tags' table.
           - If NO: INSERT new tag into 'tags' table.
           - If YES: Get existing tag_id.
           - INSERT pair (post_id, tag_id) into 'post_tags' junction table.
5. Return success & Redirect to Dashboard.

[3. Read / Dashboard]
---------------------------------------------------
1. Fetch posts from 'posts' table WHERE user_id = current_session_user.
2. Join with 'tags' to display badges.
3. Display list of cards (Title, Tags, Created_at).

[4. Edit Post]
---------------------------------------------------
1. User clicks "Edit" on a post.
2. System checks ownership (post.user_id == session.user.id).
3. Load existing data into form.
4. User modifies content/tags and saves.
5. Database Update:
   - UPDATE 'posts' table columns.
   - DELETE old relations in 'post_tags' for this post.
   - Re-INSERT new tag relations (same logic as Create Post).

[5. Delete Post]
---------------------------------------------------
1. User clicks "Delete".
2. System checks ownership.
3. Database Delete:
   - DELETE row from 'posts' WHERE id = post_id.
   - (Cascade delete will automatically remove related 'post_tags').
4. Refresh UI.
